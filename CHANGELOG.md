# Changelog

All notable changes to the AWS Troubleshooting Hub project will be documented in this file.

## [2.1.0] - 2024-12-27

### Added
- Complete networking troubleshooting section with ELB guides
- Linux system administration troubleshooting pages
- Enhanced search functionality with external AWS resource integration
- Responsive design improvements for mobile devices
- New service categories: Analytics, Big Data, Deployment, Security

### Changed
- Restructured navigation for better user experience
- Updated service categorization to match AWS support structure
- Improved accessibility with proper ARIA labels
- Enhanced CSS performance and reduced bundle size

### Fixed
- Search result highlighting and auto-expansion
- Mobile navigation menu responsiveness
- External search URL formatting for AWS re:Post and documentation

## [2.0.0] - 2024-11-15

### Added
- Multi-service architecture supporting 11+ AWS service categories
- Generic page layout component for consistent user experience
- Advanced search with auto-expansion and result highlighting
- Service-specific troubleshooting guides for SVLS and DMI

### Changed
- Complete redesign from single-service to multi-service platform
- Migrated from class components to functional components with hooks
- Updated routing structure for scalable service addition

### Removed
- Legacy single-page troubleshooting approach
- Unused CSS classes and redundant code
- Development artifacts and test files

## [1.0.0] - 2024-10-01

### Added
- Initial release of AWS troubleshooting platform
- Basic SVLS (Serverless) troubleshooting guides
- DMI (Developer Mobile, Messaging & IoT) support
- Responsive design with mobile support
- Search functionality for troubleshooting content

### Features
- Interactive tree-based navigation
- Collapsible content sections
- Smooth animations and transitions
- AWS-themed design system
